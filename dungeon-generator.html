Phase 1: Binary Space Partitioning (Divide & Conquer)

The canvas is recursively split into binary partitions until reaching minimum room size. Each leaf node becomes a room with random insets. This guarantees non-overlapping rooms by construction—no collision detection needed. Time complexity: O(n) where n is the number of rooms.

Phase 2: Prim's MST (Minimum Spanning Tree)

Connects all N rooms using exactly N−1 corridors by building a minimum spanning tree. Starts from a random room and greedily adds the shortest edge to an unvisited room. Uses Euclidean distance between room centers. Time complexity: O(E log V) with a min-heap.

Phase 3: Backtracking (Constraint Satisfaction)

Places keys and doors while enforcing the constraint: for any path from start to a locked door, the corresponding key must appear earlier. Uses DFS to verify each placement. Backtracks when a constraint is violated. Time complexity: O(k! × V) worst case, where k is number of keys.

Phase 4: Graph Analysis (DFS/BFS Verification)

Performs depth-first search to confirm all rooms are reachable from start. Detects cycles and validates key-door orderings across all possible traversal paths. BFS computes shortest path distances for difficulty balancing. Time complexity: O(V + E).
